## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-text gaps within nav items (`gap-1`) |
| sm | 8px | Compact element spacing, badge inner padding |
| md | 16px | Default element spacing, nav item horizontal padding |
| lg | 24px | Section group padding within sidebar |
| xl | 32px | Card and panel internal padding |
| 2xl | 48px | Major section breaks between nav groups |
| 3xl | 64px | Page-level vertical spacing |

Exceptions:
- Sidebar nav item vertical padding: 12px (`py-3`) — replaces prior `py-2.5` (10px was not a multiple of 4)
- Sidebar nav item horizontal padding: 16px (`px-4`) — replaces prior `px-5` (20px was not a multiple of 4)
- Active indicator bar: left-0, top-1, bottom-1, width 4px — existing pattern, preserve exactly
- Section group headers: `px-4 pt-4 pb-1` — new pattern for NAV-01 group headers

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `hsl(210 40% 98%)` — `--background` | Page background, main content area |

## Navigation Architecture (NAV-01, NAV-02, NAV-03)

### Sidebar Structure After Phase 6

The flat `navItems` array in `AppLayout.tsx` is replaced with a grouped structure. Four named sections replace the ungrouped list.

```
[Group: Overview]
  Dashboard               → /

[Group: Automations]
  All Automations         → /alle
  New Automation          → /nieuw
  Verification            → /verificatie
  Imports                 → /imports

[Group: Systems & People]
  (placeholder — populated in Phase 7)

[Group: Analysis]
  Processes               → /processen
  Analysis                → /analyse
```

### Group Header Visual Contract

- Text: Section label (e.g. "Overview", "Automations", "Systems & People", "Analysis")
- Style: `text-[10px] font-semibold uppercase tracking-widest text-sidebar-foreground/40 px-4 pt-4 pb-1`
- No icon, no link, not interactive
- First group header ("Overview") has no top padding — sits directly below the brand header separator

### Dead Pages Removed (NAV-03)

The following commented-out items in `navItems` are fully deleted from the codebase:
- Mindmap (`/mindmap`)
- Kennisgraaf (`/kennisgraaf`)
- BPMN Viewer (`/bpmn`)
- Proceskaart (`/proceskaart`)

Route definitions for these paths are also removed from `App.tsx` (or equivalent router file).

### NAV-02: Nav Label = Page h1 Contract

Every sidebar nav label must be byte-identical to the `<h1>` (or primary page heading) on its destination page. The table below defines the canonical English label for each page.

| Route | Sidebar Label | Page h1 |
|-------|--------------|---------|
| `/` | Dashboard | Dashboard |
| `/alle` | All Automations | All Automations |
| `/nieuw` | New Automation | New Automation |
| `/verificatie` | Verification | Verification |
| `/imports` | Imports | Imports |
| `/processen` | Processes | Processes |
| `/analyse` | Analysis | Analysis |
| `/instellingen` | Settings | Settings |

Note: Pages that currently have no visible `<h1>` (e.g. Dashboard, Processen, AlleAutomatiseringen) must have an `<h1>` added as a visually hidden element or visible page heading to satisfy NAV-02. The top bar `<span>` that currently mirrors the nav label title must also be updated to match.

## Status Label Consistency (NAME-03)

Canonical English status values to use on every page that displays automation status:

| Status | Display Label | CSS Class |
|--------|--------------|-----------|
| Active automations | Active | `badge-status-actief` → rename to `badge-status-active` |
| Needs review (>90 days) | Outdated | `badge-status-verouderd` → rename to `badge-status-outdated` |
| Paused / turned off | Disabled | `badge-status-uitgeschakeld` → rename to `badge-status-disabled` |
| Flagged for checking | In Review | `badge-status-review` (already English, keep) |

The DB values in Supabase (`Actief`, `Verouderd`, `Uitgeschakeld`) are NOT changed — only display labels and CSS class names change. Filter logic that compares `a.status === "Actief"` etc. is updated to compare against the DB value, not the display label.

The `STATUSSEN` constant in `src/lib/types.ts` retains Dutch DB values. A display mapping function is added:

```ts
const STATUS_LABELS: Record<string, string> = {
  "Actief": "Active",
  "Verouderd": "Outdated",
  "Uitgeschakeld": "Disabled",
  "In review": "In Review",
};
```

Verification status labels (`geverifieerd`, `verouderd`, `nooit`) are internal state values and get a similar display mapping:

```ts
const VERIFICATIE_LABELS: Record<string, string> = {
  "geverifieerd": "Verified",
  "verouderd": "Outdated",
  "nooit": "Never",
};
```

## Implementation Notes for Executor

1. **AppLayout.tsx sidebar**: Replace flat `navItems` array with a `navGroups` array of `{ title: string; items: NavItem[] }`. Render a group header `<div>` before each group's items. Group headers are not Links.

2. **Route cleanup**: Search `App.tsx` (or router file) for `/mindmap`, `/kennisgraaf`, `/bpmn`, `/proceskaart` route definitions and delete them. Delete the page files if they still exist in `src/pages/`.

3. **English label replacement**: Each file in the inventory above gets Dutch strings replaced in-place. No structural refactoring — string replacement only.

4. **Status display mapping**: Add `STATUS_LABELS` map to `src/lib/types.ts`. Update every UI render of `a.status` to use `STATUS_LABELS[a.status] ?? a.status`. Do NOT change DB filter comparisons.

5. **Page h1 requirement**: Each routed page must have exactly one `<h1>` whose text matches the sidebar nav label. Pages that currently render no visible `<h1>` (Dashboard, AlleAutomatiseringen, Processen, Analyse, Imports) need a heading added. The heading may be visually hidden (`sr-only`) if the page design does not show a visible title — but it must exist in the DOM for NAV-02 compliance.

6. **Top bar span**: `AppLayout.tsx` line 133 renders the current nav item title in the top bar. This uses the navItem `title` field. Once labels are English, this updates automatically.

7. **Sidebar nav item padding**: Use `py-3 px-4` for all sidebar nav items and group headers. Do not use `py-2.5` or `px-5`.
